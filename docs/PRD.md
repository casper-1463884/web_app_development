# 智慧烹飪菜單系統 - 產品需求文件 (PRD)

## 1. 專案概述

### 背景與動機
在現代繁忙的生活中，許多人希望在家下廚以追求健康與省錢，但往往面臨「不知道要煮什麼」或「買了食材卻不知如何搭配」的困擾。現有的食譜網站雖然多，但缺乏系統性的菜單規劃與食材庫存管理。本系統旨在結合食譜管理、菜單規劃與智慧推薦，降低下廚的決策成本。

### 目標用戶
- 一般大眾：希望提升料理效率、減少外食次數的人。
- 家庭煮夫/煮婦：需要規律規劃全家菜單的使用者。
- 料理新手：需要引導式食譜與食材推薦的使用者。

### 核心價值主張
- **省時**：透過預設菜單減少每日思考的時間。
- **省錢**：根據現有食材推薦料理，減少食材浪費。
- **健康**：鼓勵在家下廚，掌控營養攝取。

---

## 2. 功能需求

本系統包含以下主要功能與使用者故事：

1.  **食譜資料庫管理**
    - **使用者故事**：作為一位使用者，我希望能夠新增、編輯與分類我的私房食譜，以便我隨時查閱料理步驟。
2.  **每週菜單規劃**
    - **使用者故事**：作為一位家庭主夫/主婦，我希望能夠在日曆上安排每週的早中晚餐，以便我提前準備食材。
3.  **個人食材庫存管理 (My Pantry)**
    - **使用者故事**：作為一位使用者，我希望記錄家裡現有的食材與保存期限，以便我知道還有什麼東西可以用。
4.  **智慧食材推薦系統**
    - **使用者故事**：作為一位不想浪費食材的人，我希望系統能根據我剩餘的食材推薦適合的食譜，以便我能清空冰箱。
5.  **購物清單自動生成**
    - **使用者故事**：作為一位採買者，我希望系統能根據我的菜單自動列出缺少的食材，以便我在超市時不會遺漏。

---

## 3. 非功能需求

### 技術限制
- **開發框架**：Flask (Python)
- **模板引擎**：Jinja2
- **資料庫**：SQLite
- **前端風格**：Vanilla CSS (精美現代感設計)

### 效能與安全考量
- **響應式設計**：需支援手機端查看菜單與食譜。
- **資料持久性**：所有食譜與規劃需妥善儲存在資料庫中。
- **安全性**：使用者帳戶需經過基本的身份驗證。

---

## 4. MVP 範圍

### Must Have (必須達成)
- 食譜的 CRUD 功能（增刪改查）。
- 基本的每週菜單規劃介面。
- 簡易的食材標籤系統。

### Should Have (應該要有)
- 個人食材庫存管理頁面。
- 根據單一食材搜尋食譜的功能。
- 購物清單生成。

### Nice to Have (未來擴充)
- 營養成分分析（熱量、蛋白質等）。
- 社群分享功能，將菜單分享給好友。
- AI 自動識別食材圖片。
